# Overview

## Current Behavior

ApplyWise has two working AI features in the FastAPI backend
(`job_extractor.py`, `candidate_profile_extractor.py`) that call Gemini with
structured output, schema validation, and retry. Everything else "AI" is
**deterministic**: the match analyzer, resume suggestions, resume draft,
roadmap, and interview prep are generated by keyword matching and templated
strings in `apps/web/src/lib/*.mjs`, run from Next.js server actions that write
directly to Supabase.

There is no shared way to run an AI feature: no record of AI runs, no provider
abstraction, no activity feed, no standard error handling, and no observability.
Each future AI feature would otherwise re-invent prompt → schema → validate →
persist → log.

## Target Behavior

A reusable backend AI-workflow foundation exists so every Period 8 feature
(US-028–US-038) runs the same way: create an `ai_workflow_runs` row, run the
workflow, validate output with Pydantic, persist the domain result and an output
snapshot, write an `activity_feed` event, and return a typed result — with a
deterministic fallback when Gemini is unavailable and a friendly error contract
when it fails.

This story ships the **infrastructure only**. It is proven end-to-end by
porting **one** existing deterministic feature — the match analyzer — onto the
foundation as the reference implementation (the full match-analyzer AI behavior
and UI is US-028). After this story, adding an AI feature means writing a prompt,
a schema, and a persistence call against `BaseAIWorkflow`.

## Affected Users

- Software engineers using ApplyWise — indirectly. No new user-facing screen in
  this story beyond the existing match generation working through the new path.
- Future agents building US-028–US-038, who inherit one boundary.

## Affected Product Docs

- `docs/product/ai-workflows.md` (reliability + privacy rules realized here)
- `docs/product/data-model.md` (`ai_workflow_runs`, `activity_feed`)
- `docs/decisions/0012-ai-workflow-standards.md`

## Non-Goals

- Building the full match analyzer UI/behavior (US-028).
- Any of the other AI features (US-029–US-038).
- Token/cost metrics, model selector, LangGraph orchestration (post-MVP).
- Changing routing to job-centric, auth provider, or design system.
